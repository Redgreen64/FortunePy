# FortunePy 🥠
**A Fortune Library for Python and Linux.**

FortunePy is a lightweight Python wrapper that bridges the gap between the classic 1970s `fortune` command and modern Generative AI. Whether you want a traditional cookie-style quote or a fresh insight generated by an LLM, FortunePy delivers it straight to your terminal—optionally wrapped in glorious ASCII art.

## 🚀 Features
- **Legacy Support:** Access the classic Linux `fortune` database.
- **AI-Powered:** Use [Ollama](https://ollama.com/) to generate unique fortunes.
- **Big Text:** Integrated `figlet` support for ASCII art banners.
- **Global Install:** One-script installer moves the module to your `site-packages`.

## 🛠️ Requirements
To use the full power of FortunePy, you'll need:

1. **Linux Utilities:**
   ```bash
   sudo apt install fortune-mod figlet
   ```
   **⚠️ CRITICAL:** The command **MUST** be accessible as `fortune` in your terminal. If your distro calls it `fortune-mod`, symlink it to `fortune` or this script will fail. I'm not writing a search algorithm for your binaries.

2. **Ollama:** [Download and install Ollama](https://ollama.com/) for AI features.
3. **Python Library:** `pip install ollama`

## 💻 Usage

### The Classics
```python
import fortunepy

# Standard terminal fortune
print(fortunepy.fortune())

# Fortune rendered in ASCII Art
print(fortunepy.ASCIIFortune())
```

### The Modern Way (AI)
```python
# Generate a fortune using an LLM
# Defaults to gpt-oss:120b-cloud, but you can specify llama3, mistral, etc.
print(fortunepy.aiFortune(temp=0.8, AImodel="llama3"))

# AI-generated fortune in ASCII Art
print(fortunepy.aiASCIIFortune(temp=1.0, AImodel="mistral"))
```

## 📖 API Reference

| Function | Description |
| :--- | :--- |
| `fortune()` | Returns a standard string from the system `fortune` command. |
| `aiFortune(temp, AImodel)` | Generates a fortune via Ollama. |
| `ASCIIFortune()` | Returns a standard fortune piped through `figlet`. |
| `aiASCIIFortune(temp, AImodel)` | Generates an AI fortune and renders it in `figlet`. |

## FAQ

**Q: Why are you using `os.popen` and double quotes for shell commands?**  
**A:** Because it works. If you disagree... look in the code.

**Q: Why only Linux?**  
**A:** Because I don't have the source to `fortune`. Use WSL if you really need this on Windows.

**Q: It says "command not found" even though I installed it!**  
**A:** Your distro named the binary `fortune-mod` or `bsd-fortune`. My code looks for `fortune`. I’m not changing it. Link the binary to `/usr/bin/fortune`.
